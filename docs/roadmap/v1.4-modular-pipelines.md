# Creation Station v1.4 - Modular Pipelines

## Goal

Introduce a configuration-first modular pipeline architecture without adding separate app systems.

Core workflow remains:

```text
Idea Inbox -> AI Factory Planner -> Review Inbox -> Revision -> Approval -> Tasks
```

## Architecture

v1.4 adds a shared pipeline registry for:

- Music pipeline
- Visual pipeline
- Game pipeline
- Automation pipeline
- General planning fallback

The registry owns pipeline names, route keywords, category aliases, Factory prompt guidance, default labels, recommended sections, and release checklist text.

## Rules

- Do not add new routes for individual pipelines.
- Do not add schema fields in v1.4.0.
- Do not add agents, connectors, automation engines, generated asset systems, or external runners.
- Keep pipeline behavior as configuration consumed by shared systems.
- Preserve export backup compatibility.

## Shared Systems

The existing shared systems remain authoritative:

- Idea routing
- Factory prompt construction
- Review and revision
- Task generation
- Task board filtering and grouping
- Task dependencies through `TaskBlocker`
- Intelligence recommendations
- Export backup

## Implementation Slice

v1.4.0 should include:

- `src/lib/pipelines.ts`
- route detection refactor to consume pipeline definitions
- Factory prompt guidance by detected pipeline
- compact pipeline filters and counts
- no Prisma migration

## Acceptance

- Music, visual, game, and automation sample ideas route correctly.
- Factory plan JSON shape is unchanged.
- Approved plans still create tasks normally.
- Task board remains structurally unchanged.
- Lint, type check, Prisma validation, build, and localhost smoke pass.
