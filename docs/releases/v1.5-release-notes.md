# Creation Station v1.5 Release Notes

Release candidate: `v1.5.0-rc1`

Branch: `feature/v1.5-release-readiness`

## Release Summary

Creation Station v1.5 is a release-readiness milestone for the existing local-first solo creator workflow:

```text
Idea Inbox -> AI Factory Planner -> Review Inbox -> Revision -> Approval -> Tasks
```

This release focuses on making the current workflow easier to understand, safer to operate, easier to back up, and more useful during day-to-day execution. It does not add cloud sync, accounts, teams, external connectors, agents, or v2 architecture.

## Features Added

- Task Board scalability:
  - separated Active, Blocked, Backlog, Completed, and Archived task areas
  - kept Active and Blocked highly visible
  - kept Completed and Archived collapsed by default
  - added project filtering alongside existing task filters

- First-use onboarding:
  - added a lightweight welcome flow for fresh workspaces
  - explains Idea Inbox, project planning, task execution, and the core workflow
  - supports skip/completion with browser-local persistence
  - provides quick actions for first idea and AI setup

- Manual backup and restore:
  - added v1.5 JSON export shape with version metadata
  - added manual import and restore controls in Settings
  - added restore confirmation and invalid-backup rejection
  - preserved compatibility fields for older export consumers

- Local analytics:
  - added local-only event logging for operational workspace events
  - added Settings analytics summary for task creation, completion rate, idea conversion, project creation, backups, onboarding state, and recent events
  - stores events in ignored local app data with retention capped to the last 500 events

## UX Improvements

- Idea Inbox:
  - improved filter toolbar layout and Apply button placement
  - added clearer visual hierarchy for idea cards
  - added derived visual states: New, Reviewing, Converted, Archived
  - added plan/task counts and latest plan status
  - bounded long idea lists to reduce scrolling clutter

- Task Board:
  - improved status grouping and summary counts
  - added a project filter based on existing plan data
  - improved empty guidance for first task creation

- Empty states:
  - clarified first idea, first project plan, and first task paths
  - kept guidance tied to the existing workflow instead of adding new systems

## Reliability Improvements

- Backup validation rejects invalid or empty backup files safely.
- Restore requires both browser confirmation and typed `RESTORE` confirmation.
- Export includes ideas, projects, tasks, task blockers, settings metadata, and version metadata.
- Analytics endpoint rejects unsupported client-side event types.
- Local analytics retention prevents unbounded event-log growth.
- Release QA verified route health, export shape, invalid restore, valid restore, fresh-workspace rendering, analytics logging, and analytics retention.

## Backup System

Export format includes:

```json
{
  "version": "v1.5",
  "appVersion": "0.1.0",
  "exportedAt": "...",
  "generatedAt": "...",
  "settings": {},
  "ideas": [],
  "projects": [],
  "factoryPlans": [],
  "tasks": [],
  "taskBlockers": []
}
```

Notes:
- `projects` represents Factory plans in the current data model.
- `factoryPlans` remains included for compatibility.
- Unknown backup fields are ignored.
- Invalid core records are rejected.
- Restore replaces local workspace data by design.

## Analytics System

Tracked local events:

- `idea_created`
- `idea_converted`
- `project_created`
- `task_created`
- `task_completed`
- `task_archived`
- `backup_exported`
- `backup_imported`
- `onboarding_completed`
- `onboarding_skipped`

Analytics are stored locally only. No external analytics tools, tracking pixels, accounts, or cloud services are used.

## Known Limitations

- Project edit, task edit, and idea edit controls are not implemented in v1.5.
- Delete project/task/idea flows are not implemented; archive/status transitions are the supported safe paths.
- Restore is intentionally destructive and should be used only after exporting a backup.
- Analytics are lightweight local file writes and are not designed for concurrent multi-user or multi-process usage.
- Browser-local prompt presets are not embedded as full export data; the backup records the settings mode.
- Full onboarding skip/completion browser click-through should be repeated on a disposable profile before final stable tagging.

## Deferred To v2

- user accounts, authentication, permissions, and teams
- real cloud sync, background sync, conflict resolution, and multi-device state
- external storage providers or connector systems
- hosted deployment architecture
- event-sourced analytics or historical reporting infrastructure
- agents, automation runners, plugin systems, meetings, calendars, and full asset vault expansion
- major Prisma redesign or new workflow subsystems
