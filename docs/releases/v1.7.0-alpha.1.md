# Creation Station v1.7.0-alpha.1

Release name: `Creation Station v1.7.0-alpha.1`

## Summary

This alpha release packages the new execution-layer groundwork for Creation Station. It adds a bounded autonomy preview surface, persistent worker queue infrastructure, music video execution flows, worker-facing APIs, and supporting documentation for the v1.7 autonomy model.

## Added

- Execution Layer route and UI for music video pipeline planning and package generation.
- Music video builder APIs for queuing, status polling, and release asset downloads.
- Execution request queue, worker runtime tracking, and daemon entrypoint for local worker processing.
- Autonomy execution-kernel preview modules, sandbox validation, approval policy, tool adapters, live unlock checks, and worker monitoring.
- New autonomy and worker test coverage for queueing, sandbox enforcement, live unlock, rollback, and worker recovery behavior.
- Autonomy architecture documentation covering kernel flow, memory model, failsafes, human override, and test scenarios.

## Changed

- Inbox and dashboard surfaces now expose execution-layer status and worker runtime visibility.
- Factory and execution actions now support music video pipeline creation and sample worker queue actions.
- Creative execution packaging now emits a fuller release bundle including final video, thumbnail, workflow, metadata, prompt text, and manifest files.
- Navigation now includes a dedicated Execution Layer entry.
- `.gitignore` now ignores the local `output/` directory.
- `package.json` now includes the `worker:daemon` script.

## Fixed

- Music video builder validation now rejects missing audio and empty workflow submissions early.
- Builder error handling now reports failed ComfyUI runs and FFmpeg merge failures through tests.
- Worker execution flow now covers stale-claim recovery, rollback after failed file writes, and persisted worker shutdown state.
- Execution-kernel preview now deduplicates approval requests, blocks malformed or dangerous actions, and rejects stale planned state.
